Global Component Lumping for EOS Calculations

Authors: Seyyed Ali Alavian, Curtis H. Whitson, Snjølfur Ørn Martinsen (PERA / NTNU)

Venue: SPE Annual Technical Conference and Exhibition, Amsterdam, 27-29 October 2014

Paper number: SPE-170912

Citations: not catalogued

EOS Lumping Pseudoisation Combinatorial Search Weighted RMS MMP Filter PhazeComp Pipe-It Streamz C7+ Phase 4

Context

Ali Alavian previously authored CO2-EOR and diffusion-in-fractured-reservoir papers with Curtis (2009, 2010). Snjølfur Martinsen co-authored the Pauto Colombia multi-compartment EOS characterisation (Martinsen et al. 2010). The 2014 paper transforms component lumping for EOS pseudoisation from an empirical, heuristic task into a systematic automated global search via three ingredients: combinatorial enumeration of contiguous-lumping scenarios, weighted-RMS quality measurement against a detailed reference EOS rather than fitting to measured data, and a two-stage minimum-miscibility-pressure (MMP) filter that adds miscible-process accuracy without exploding CPU time. Workflow is implemented via Whitson's PhazeComp (Zick Technologies) EOS engine and Pipe-It / Streamz (Petrostreamz) model automation. Three worked examples reduce an EOSxx(34) PR-EOS plus Lohrenz-Bray-Clark (LBC) model to EOSx(15), EOSx(9), and EOSx(6) lumped models.

Key Contributions

  • Combinatorial enumeration of all feasible contiguous-lumping scenarios. The combinatorial formula NEOSx = (Nxx-1)! / [(Nxx-Nx)!(Nx-1)!] gives the number of distinct contiguous-lumping scenarios that lump Nxx source components into Nx pseudo-components. For EOSxx(34) reduced to EOSx(15), EOSx(9), EOSx(6) the totals are 8,855 / 142,506 / 237,336 scenarios respectively - tractable by overnight batch. Three optional user-configurable lumping constraints prune the space: light-pure-component restrictions (N2 only with C1, CO2 only with C2, H2S only with C3), contiguous-only hydrocarbon lumping ordered by normal boiling point (NBP), and no unconstrained partial lumping (each component placed entirely into one pseudo-component). The Ruby subset-generation algorithm by Matsumoto (1993, open-source) handles enumeration.
  • Weighted-RMS quality against detailed-reference EOS, not measured data. The methodological innovation that makes the approach work. The lumped EOSx is compared not against measured laboratory data but against ~1,200 simulated PVT data points generated by the detailed EOSxx across the relevant pressure-temperature-composition space, covering five reservoir fluids (lean gas-condensate, rich gas-condensate, near-critical oil, more-volatile oil, less-volatile oil). This gives the monotonic-decrease-with-Nx property that is theoretically guaranteed but violated when fitting to measured data (because measurement scatter creates spurious local minima in the lumped fit).
  • Two-stage MMP filter for miscible-process accuracy. MMP computation is 10-20x slower than a full PVT suite, so computing MMP for all 105 combinatorial scenarios would be intractable. The two-stage workflow: (1) compute weighted RMS for all scenarios (~minutes per scenario, parallelisable); (2) compute MMP for only the 100-500 lowest-RMS scenarios. Result for example 1: EOSx(9) best-RMS scenario achieves 1.265% RMS and best-MMP scenario #34,458 has MMP within 0.4 psi of the detailed model while retaining less than 2% RMS. For EOSx(6): best-MMP scenario achieves 0-psi MMP match with 3.75% RMS.
  • Unexpected empirical finding: heavy-fraction lumping helps MMP accuracy. Best-MMP EOSx(9) and EOSx(6) scenarios for Examples 2 and 3 did not preserve C30+ as the heaviest fraction but instead lumped C30+ with C24+ to C26+. By contrast, all five best-RMS EOSx(15) scenarios did preserve C30+. The implication: some accuracy loss from light-fraction lumping is compensated by heavy-fraction lumping for MMP accuracy. This contradicts the conventional "keep the heaviest fraction intact" heuristic and could only have been discovered by exhaustive search.

Significance and Contribution

Automated systematic search beats expert heuristics. Curtis's teaching pattern: component lumping has been an art form for decades (Whitson 1983, Coats 1985, stepwise regression workflows). The 2014 paper turns it into engineering. Combinatorial enumeration with PhazeComp + Pipe-It / Streamz orchestration evaluates ~105 scenarios overnight. The right scenario emerges from the search; the engineer's role becomes specifying constraints and validating the result rather than guessing the answer.

Reference-EOS evaluation as the right error metric. The choice to evaluate lumped models against the detailed reference EOS rather than measured laboratory data is non-obvious but load-bearing:

The lumped EOS x model is compared not against measured data but against ~1200 simulated PVT data points generated by the detailed EOS xx model across the relevant pressure-temperature-composition space.

This separates two error sources: (a) the lumping approximation, and (b) the underlying EOS fit to measurements. Mixing them gives spurious local minima where the lumped EOS happens to compensate the detailed EOS's measurement-fit errors. Separating them yields the monotonic-decrease-with-Nx property that makes the search interpretable.

The unexpected heavy-fraction-lumping result. Exhaustive search produces discoveries that expert heuristics miss. The finding that lumping C30+ with C24+-C26+ improves MMP accuracy contradicts conventional wisdom and would not have been generated by any rule-based workflow. Worth noting in any future review of pseudoisation practice.
